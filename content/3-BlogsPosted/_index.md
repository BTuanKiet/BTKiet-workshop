This section introduces the blogs that have been published on the AWS Study Group community, covering the latest AWS technologies, architecture best practices, and cloud-native solutions.

### [Blog 1 - Better Together: Amazon EKS Auto Mode and Istio Ambient Mesh](3.1-Blog1/)
This blog introduces how Amazon EKS Auto Mode and Istio Ambient Mesh work together to simplify Kubernetes operations. It explains how EKS Auto Mode automates infrastructure management while Istio Ambient Mesh provides secure service-to-service communication through a sidecarless architecture, reducing operational overhead and improving scalability.

### [Blog 2 - Deploying Multi-Turn RL Infrastructure for Amazon Nova on Amazon SageMaker HyperPod](3.2-Blog2/)
This blog presents the reference architecture for deploying Multi-Turn Reinforcement Learning (RL) infrastructure using Amazon SageMaker HyperPod and Amazon Nova. It explains how AWS combines an event-driven architecture, automated infrastructure provisioning, and reward-based training to efficiently train AI agents capable of handling complex multi-step tasks.

###  [Blog 3 - ...](3.3-Blog3/)
This blog introduces the newly added session policies feature in Amazon EKS Pod Identity, which allows you to narrow IAM permissions flexibly and precisely for each pod without needing to create multiple separate IAM roles. This is an important step forward that helps apply the principle of least privilege more effectively in large-scale Kubernetes environments.
